## 1. @ai_9684xtpa (Ai 姨) - 链上数据分析方法论

### 核心方法总结

Ai姨是国内最知名的链上数据分析师之一，她的核心方法论是**"链上数据追踪 + 实时清算监控"**，通过追踪鲸鱼钱包、机构地址的资金流动来获取市场信号。

### 核心方法论要点

#### 1.1 链上地址追踪体系
- **鲸鱼地址监控**：持续追踪大户（如7 Siblings、易老板Trend Research）的持仓变动
- **交易所资金流向**：监控大额充值/提现行为，判断买卖意图
- **关联地址分析**：通过资金关联发现隐藏的地址关系

#### 1.2 清算价格计算工具
- 推荐工具：**DeBank**、**Aave/Maker清算监控工具**
- 核心指标：健康度(Health Factor)、清算价(Liquidation Price)、抵押率
- 实战案例：易老板6个杠杆地址的清算区间实时监控 [$1,574 - $1,681]

#### 1.3 数据验证流程
1. 发现大额转账 → 2. 确认地址归属 → 3. 计算成本/盈亏 → 4. 评估市场影响

### 可操作的步骤

**步骤1：建立监控清单**
```
- 选定10-20个重点监控地址（鲸鱼、机构、知名项目方）
- 使用DeBank收藏地址，开启价格预警
- 关注交易所大额充提通知
```

**步骤2：学习使用基础工具**
```
- DeBank：查看地址总资产、持仓分布
- Arkham：地址标签和关联分析
- Dune Analytics：自定义链上数据查询
- 借贷协议前端：直接查看清算数据
```

**步骤3：建立分析框架**
```
- 资金流入交易所 → 可能卖出（利空）
- 资金从交易所提出 → 可能持有（利好）
- 接近清算价 → 关注强制卖出风险
- 大额链上买入 → 关注支撑位形成
```

**步骤4：输出标准格式**
```
钱包地址：https://debank.com/profile/0x...
过去X小时：动作（买入/卖出/充值/提现）

当前盈亏：$X（X%）

```

## 2. @0xSleepinRain (雨中狂睡) - 叙事分析与"铲子理论"

### 核心方法总结

雨中狂睡是国内顶级的叙事分析师，核心方法论是**"铲子理论"**——在新叙事爆发期，优先投资"卖铲子"的基础设施项目，而非直接押注具体应用。

### 核心方法论要点

#### 2.1 铲子理论框架
在新叙事中，将项目分为三类：
- **铲子（基础设施）**：launchpad、框架、协议层
  - 例：$BNKR、$CLAWNCH（Agent发币平台）
  - 特点：无论哪个Agent成功，平台都能收取费用
- **镐头（工具/服务）**：为叙事提供工具的项目
  - 例：$CLAWD（Agent工具）
  - 特点：服务铲子的配套项目
- **淘金者（具体应用）**：具体的应用/代币
  - 例：各种Agent代币、Meme币
  - 特点：成功率低，但成功回报高

#### 2.2 叙事生命周期判断
- **早期（0-1）**：概念提出，基础设施稀缺 → 买铲子
- **成长期（1-10）**：生态扩张，项目涌现 → 持有铲子+精选应用
- **成熟期（10-100）**：竞争加剧，分流严重 → 关注头部应用
- **衰退期**：叙事消退 → 退出或寻找新叙事

#### 2.3 生态位分析
```
类比框架：
- Moltbook = 框架（Agent社交平台）
- Clawnch = Virtuals（Agent发币平台）
- Clawd = aixbt（独立Agent工具）

生态位判断：
- $CLAWNCH由人推动 → Base/Coinbase认可的核心资产
- $CLAWD由Agent自己推动 → 持续为代币赋能
- $MOLT是叙事核心Meme → 与情绪和注意力强相关
```

### 可操作的步骤

**步骤1：识别新叙事**
```
- 关注头部KOL的共识转向
- 观察是否有新的协议/标准推出（如ERC-8004、x402）
- 追踪顶级VC的投资方向
- 监测Social Graph上的热点话题
```

**步骤2：分析生态系统**
```
- 绘制叙事生态图：基础设施 → 中间件 → 应用层
- 识别"铲子"项目：谁在为整个生态提供基础设施？
- 评估竞争格局：同类铲子项目的优劣对比
```

**步骤3：建立仓位策略**
```
- 70%仓位：铲子项目（稳健）
- 20%仓位：优质镐头（平衡）
- 10%仓位：投机淘金者（高风险高回报）
```

**步骤4：跟踪叙事演变**
```
- 监控关键指标：发币数量、TVL增长、交易量
- 关注分流风险：新项目的资金分流效应
- 评估团队交付：开发者的活跃度和交付质量
```

### 实战案例（从推文提取）

**案例：OpenClaw生态分析**
- 判断：OpenClaw代表Agent+Crypto的新叙事
- 铲子识别：$CLAWNCH（Agent发币平台）、$BNKR（基础设施）
- 分析：Coinbase Jesse的背书 → $CLAWNCH是Base认可的核心资产
- 操作建议：回调到筹码密集区是买入机会
