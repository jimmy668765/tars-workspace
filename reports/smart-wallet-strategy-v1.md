# 聪明钱包发现与监控系统方案

## 1. 数据来源策略

### 方案 A: DEX Screener (待解决 Cloudflare)
- 端点: `api.dexscreener.io/latest/dex/pairs/{chain}`
- 筛选条件:
  - 市值 < $100k
  - 24h 交易量 > $10k
  - 24h 价格变化 > 50%
  - 早期买入标记 (<1小时)

### 方案 B: Solscan/Birdeye API (推荐)
- Birdeye API: `public-api.birdeye.so`
- 提供免费 tier，支持钱包追踪
- 可获取历史收益率数据

### 方案 C: 链上直接分析
- 使用 Solana RPC 获取交易数据
- 分析 Token 发射后的早期买家
- 计算收益率并分级

## 2. 聪明钱包分级标准

### Tier 1 (核心跟踪)
- 30天收益率 > 1000%
- 胜率 > 70% (10笔以上交易)
- 单笔盈利 > 100%
- 交易频率适中 (不过度 FOMO)

### Tier 2 (观察)
- 30天收益率 300-1000%
- 胜率 50-70%
- 有爆发潜力但波动大

### Tier 3 (试验)
- 胜率不稳定
- 偶尔抓到大机会
- 可作为反向指标

## 3. 监控触发条件

不是每笔交易都跟，而是:
- ✅ 单个标的同时被 ≥3个 T1 钱包买入
- ✅ 买入金额 > $1000
- ✅ 标的市值 < $10万
- ✅ 24h 内新闻/叙事匹配

## 4. 双轨监控架构

```
信息流监控 (先手)
  ↓
Twitter 热点 → 叙事分析 → 早期机会
  ↓
链上监控 (验证)
  ↓
聪明钱包异动 → 交叉验证 → 执行信号
```
