## 高质量信息定义

### 真正有用的信息 = 可转化为行动

| 类型 | 举例 | 行动 |
|------|------|------|
| **宏观判断** | "BTC将到蓝色支撑带" | 调整仓位/止损 |
| **新工具** | "VISI0 AI视频上线" | 试用/集成 |
| **Alpha** | "某协议TVL暴增原因" | 研究/进场 |
| **风险预警** | "某项目合约有漏洞" | 撤资/对冲 |
| **方法论** | "期权抄底策略" | 学习/执行 |

### 垃圾信息特征
- 纯情绪输出（"比特币要归零了！"）
- 无数据支撑的观点
- 已经priced in的新闻
- 无法验证的"内幕"

---

## 监控策略

### 信息流先手 (Twitter)
1. Tier S 实时监控 → 立即推送
2. Tier A 每日汇总 → 早晨简报
3. 关键词监控: "空投" "主网上线" "融资" "漏洞"

### 链上后手验证
1. 发现Tier S提及的标的 → 查链上数据
2. 多个Tier A同时提及 → 重点关注
3. 新币发射 → 查早期买家

### 质量控制
- 每条信息必须回答: "So What? 能做什么?"
- 不能转化为行动的 = 不推送
- 猜测性内容 = 标记"待验证"
